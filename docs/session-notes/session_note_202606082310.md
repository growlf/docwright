## Summary

Traced and fixed the silent ⚡ Review button failure — the plan-review endpoint was calling `/session/:id/prompt` (returns HTML) instead of `/session/:id/message` (returns JSON), causing a parse error that was swallowed silently. Improved PlanReviewPanel to show the error text when `findings` starts with "Error:" rather than always falling back to "No review yet." Fixed the proposal approval UX — after approving a proposal the user was dropped onto a bare template with no feedback; added a `?from=proposal` URL signal that triggers auto-improve immediately after the plan loads. Filled three bare template plans (bundle-chat-session-panel, sub-plan-parent-tracking, research-plan-execution-modes) with full technical detail and improved two of them with specific API shapes, component names, algorithm steps, and named test functions.

## Decisions Made

- `?from=proposal` URL param is the signal for auto-improve-on-load (cleaner than session storage)
- `pendingAutoImprove` flag + sticky toast pattern matches the existing `showImproveTab` approach
- Plan review endpoint stays synchronous (wait for full OpenCode response) then streams back progressively — consistent with the improve endpoint
- YAML titles with colons must be quoted in frontmatter (enforced by hook's `yaml.safe_load`)
