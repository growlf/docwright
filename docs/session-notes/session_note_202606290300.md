## Summary

Implemented the complete `chat-architecture-document-scoped-sessions` plan (6 steps).
ChatPanel now maintains a `Map<filePath, sessionId>` in localStorage so each document
gets its own persistent chat history. Stale sessions auto-recover on OpenCode restart.
A `📄 filename` indicator and `↺` new-chat button show in the header. Defined 4 typed
specialist AI roles in `ai-roles.ts` (doc-reviewer, doc-improver, plan-executor,
doc-assistant) and wired them into plan-review and apply-review routes.
Added `/api/ai-specialist` endpoint and `bridge.aiSpecialist()` / `aiSpecialistStream()`
/ `aiRoles` so plugins get high-quality AI without managing OpenCode themselves.
Documented in `docs/plugins.md`. e2e tests written (`npm run test:chat-sessions`).

Also: reverted naive document content dump into chat messages; captured and improved
the architecture proposal; cancelled the two superseded partial plans.

## Decisions Made

- Document-scoped sessions use `dw-chat-sessions` localStorage key, capped at 20 LRU
- Stale recovery: 404/410 → clear + recreate silently; network errors → preserve session
- `ai-roles.ts` lives in `$lib/` (not `$lib/server/`) so it's usable client + server
- `aiSpecialistStream()` is single-emission now; real streaming deferred to plan-executor work
- System prompt injected as `role: 'system'` best-effort after session creation
