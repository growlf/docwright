## Summary

Diagnosed the erp-images plugin outage: the Web UI Vite process was crashing because /api/watch's recursive fs.watch on the vault root raced with gitstatusd temp dirs under .git/, emitting an unhandled FSWatcher 'error' event. Patched the watch endpoint to attach an error handler and ignore .git/ events, restarted DocWright serving the cs-erp-images vault, and verified the plugin API responds and the server survives .git churn. Cherry-picked the fix onto main; direct push was rejected by branch protection, so opened PR #121 from fix/watch-endpoint-crash.

## Decisions Made

- Watcher errors are reported over the SSE stream and close the watcher instead of crashing the process
- .git/ events are filtered out of the watch stream entirely
- Fix goes to main via PR #121 (branch protection requires it); duplicate commit remains on docs/three-instance-deployment
