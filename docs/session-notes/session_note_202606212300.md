# Session Note: 2026-06-21 — Launcher Portability, MCP Vault Fix, Security Patches

**Date:** 2026-06-21
**Author:** NetYeti @ phoenix
**Focus:** Launcher portability + MCP vault wiring + security patches

## Summary

Session focused on making the DocWright launcher vault-aware and fixing a significant root cause of OpenCode MCP failures. The `DocWright` launch script was updated to use `$PWD` as the vault root by default, and `DOCWRIGHT_VAULT_ROOT` is now exported so OpenCode's `dw-vault` MCP server targets the correct vault. The hardcoded CascadeSTEAM path in the global OpenCode config was removed and the stale `dw-mcp` SSE entry cleaned up. The launcher also gained idempotent adopt/upgrade on every launch, was moved into the repo at `scripts/DocWright`, and symlinked from `~/.local/bin`. All managed vaults (csdocs, bms-ai-cluster, DAFO) were upgraded to current DocWright requirements and pushed. Dependabot vulnerabilities (2 high, 2 medium, 1 low) were resolved via npm overrides without breaking changes.

## Decisions Made

- `DOCWRIGHT_VAULT_ROOT` flows from shell env into OpenCode MCP via `process.env` spread — hardcoded path removed from global opencode config
- Idempotent adopt/upgrade runs on every `DocWright` launch (manifest hash baseline — fast no-op when current)
- Launcher stored in `scripts/DocWright` and symlinked; `REPO` derived from script location
- Stale `dw-mcp` SSE entry (port 3001) removed from global OpenCode config
- npm `overrides` chosen over `--force` for security fixes to avoid breaking direct dep ranges
