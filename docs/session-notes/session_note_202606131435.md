# Session Note: 2026-06-13 — Plan review holistic path + step generation

**Date:** 2026-06-13
**Author:** NetYeti @ phoenix
**Focus:** Holistic plan review for no-steps plans

## Summary

Implemented the holistic review path in `/api/plan-review` when a plan has no Implementation Steps defined. Instead of returning an error, it now runs a single prompt analyzing the plan's core goal, suggesting 3-5 concrete steps, identifying approach gaps, and noting preconditions. On Apply, `/api/apply-review` generates an initial Implementation Steps table from that analysis and inserts it into the plan. After Apply, the Review tab stays active (instead of switching to Properties) so the user can immediately run a full per-step review on the now-populated steps. Touchscreen mode badge click handling was also fixed (X/Y coords check). Build passes.

## Decisions Made

- When no Implementation Steps exist, plan-review does a holistic analysis (single prompt) instead of erroring out
- apply-review generates a steps table from the holistic analysis when no steps exist
- After step generation Apply, stay on Review tab so user can optionally re-review with steps now populated
- generatedSteps flag returned from apply-review API to let the UI decide tab behavior
