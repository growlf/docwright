# Session Note: 2026-06-14 — Olla GPU routing fix

**Date:** 2026-06-14
**Author:** NetYeti @ phoenix
**Focus:** Olla GPU routing fix

## Summary

Investigated and fixed AI inference routing to offload LLM work from local Intel Arc (CPU fallback/thermal) to remote NetBird Nvidia node. Identified root cause of crashes: Intel Arc integrated GPU has 0 dedicated VRAM, causing 14B models to exhaust system memory. Changed Olla load balancer from `least-connections` to `priority`, bumped remote Nvidia node priority to 90. Reduced local ollama Docker mem_limit from 32g to 8g. Updated opencode.jsonc to route through Olla OpenAI-compatible endpoint. Created research doc with full findings. Updated `intel_nuc_skullcanyon_ollama_with_gpu` repo with tool-calling guidance, memory optimizations, and Olla routing docs.

## Decisions Made

- Use `priority` load balancer (not `least-connections`) so Olla routes by endpoint priority
- Remote Nvidia node at priority 90, local Arc at 70
- Default to `llama3.1:8b` — only model reliably generating tool calls through Olla
- Local ollama Docker capped at 8g memory to prevent OOM/swap storms
