# Session Note: 2026-06-14 — Phoenix Arc GPU acceleration

**Date:** 2026-06-14
**Author:** NetYeti (growlfd@gmail.com) @ phoenix
**Focus:** Phoenix Arc GPU acceleration + local AI reliability

## Summary

Fixed GPU acceleration on Phoenix (Meteor Lake-P Intel Arc Graphics) by replacing the broken ipex-llm/SYCL Docker ollama with native Ollama v0.30.7 + Vulkan backend. Vulkan detects the iGPU with 23 GiB shared memory, offloads 25/25 layers for small models and 26/49 for the 14B coder model. Created `ollama-native.service` systemd unit with `OLLAMA_VULKAN=1` and `OLLAMA_IGPU_ENABLE=1`. Removed Docker ollama from ai-stack docker-compose. Updated knowledge repos with Meteor Lake Arc Vulkan path.

## Decisions Made

- **Rejected** ipex-llm/SYCL Docker path for Meteor Lake Arc (SYCL reports 0 VRAM on UMA shared memory, project archived Jan 2026)
- **Chose** native Ollama + Vulkan path — tested non-destructively on port 11435 first
- Added `OLLAMA_IGPU_ENABLE=1` — required because Ollama v0.30+ drops integrated GPUs by default
- Removed Docker ollama from `docker-compose.arc.yml` — native ollama serves port 11434 directly
- Replaced system `/usr/local/bin/ollama` with symlink to native v0.30.7 binary
