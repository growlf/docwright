## Decisions Made

- **Governance Engine replaces Policies VC** — DocWright is a policy execution engine,
  not a document manager with a policies section. Primary activity bar slot (order: 10).
- **Multi-user auth is critical and pre-emptive** — must land before API surface grows.
  `hooks.server.ts` + Forgejo OAuth + OCC for concurrent writes. Priority: critical.
- **Branch-per-component model** — each VC/plugin gets its own branch tracking develop,
  allowing independent timelines while contributing to DocWright milestones.
- **Plugin lazy-loading** — external plugin bundles load on first activation click, not
  at startup. Core VCs register eagerly in layout `onMount`.
- **`vcRegistryVersion` store** — solves the timing race where `ViewContainerMount.$effect`
  fires before `layout.onMount` has registered core VCs.
- **`lazy` prop on ViewContainerMount** — core VCs skip bundle-load attempt, preventing
  the "Plugin error" flash that occurred before the VC registered.
- **`coreVCs.ts`** — all core VC registrations isolated from layout, making
  `+layout.svelte` fully view-agnostic.
