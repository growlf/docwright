**Focus:** Review button SSE streaming fix

## Summary

Fixed the Review button UX so it immediately switches to the review tab with live streaming of the AI's response. Redesigned the review from a passive critique into actionable improvements that can be accepted and applied. Root cause was a client-side SSE parser bug — it checked `data.type` instead of the SSE `event:` field, so every `token`, `status`, and `done` event was silently discarded.

## Decisions Made

- Store-signal pattern (`showReviewTab` writable + `$effect` watcher) instead of inline `rightTab = 'review'` — matches proven `showImproveTab` / `showRelatedTab` pattern
- SSE streaming from server with `event:` type dispatch rather than `data.type` — conforms to SSE spec, avoids silent event drops
- 80-char chunks with 15ms inter-chunk delay balances visible streaming effect against total display latency
- AI prompt asks for `=== CHANGES ===` and `=== IMPROVED PLAN ===` sections — parseable without fragile JSON generation
- Removed `await tick()` from handlers — unnecessary when tab switch is driven by reactive store + `$effect`
