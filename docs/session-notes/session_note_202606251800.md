**Date:** 2026-06-25
**Author:** NetYeti @ phoenix
**Focus:** DocWright-kg environment setup, knowledge graph UI, device scanner architecture

## Summary

Long session focused on three areas: getting the DocWright-kg branch running as a testable environment (symlinks, npm installs, launcher fixes, adopt-vault bugs), implementing and iterating on the KG graph views in the bms-ai-cluster vault (ForceGraph shared component, uplink topology), and a deep scanner refactor in bms-ai-cluster to fix root-cause data quality issues (duplicate devices, missing relationships, scan-order dependency bugs).

The biggest architectural win was the gather‚Üíresolve‚Üíwrite pipeline (`DeviceRegistry`) replacing the old sequential scan-and-write approach. This eliminated the class of bug where scan order determined relationship correctness (e.g. `yc-office-switch` pointing to itself).

Playwright e2e checks are now committed as a permanent verification tool ‚Äî the `npm run test:e2e` gate enforces that UI changes are verified before being reported as working.

## Decisions Made

- **`DocWright-kg` symlink** created at `~/.local/bin/DocWright-kg` pointing to the KG branch launcher
- **Launcher `--vault` flag** added to all three DocWright repos ‚Äî eliminates stale `DOCWRIGHT_VAULT_ROOT` env var footgun; `$PWD` is always the default
- **`adopt-vault.ts` self-stomp fix** ‚Äî skip writing `.claude/settings.json` when `dest === docwrightPath`
- **Playwright mandatory** ‚Äî never report UI correctness without running `npm run test:e2e` first; Stop hook enforces this
- **erpnext/technitium removed** from all three DocWright repo configs ‚Äî belonged only in bms-ai-cluster
- **ForceGraph.svelte** created as shared D3 component; `KnowledgeGraph` and `GraphView` refactored to thin wrappers
- **`DeviceRegistry` gather‚Üíresolve‚Üíwrite pipeline** ‚Äî MAC is ground truth identity; resolution happens after all sources gathered; validation before any writes
- **`uplink_host` is a first-class field** ‚Äî in `ref_fields` (renders graph edges), `extra_fields` (in device notes), YAML sources (survives syncs); NOT in `YAML_DERIVED` (enrich-uplinks values preserved)
- **Duplicate devices stay visible** ‚Äî scanner reports them, never auto-merges; human review required
