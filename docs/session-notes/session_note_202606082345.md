## Summary

Implemented plan-to-phase assignment end-to-end: PropertiesPane dropdown, create-plan auto-detect, linter validation, status page phase grouping, and backfill of all 8 unassigned plans. Added an AI model picker in the toolbar that reads OpenCode's live `/api/model` list (112 models) and writes the selection to `opencode.json` — replacing a failed attempt at custom Ollama integration that hit env-var and session-API obstacles. Fixed the auto-improve-on-new-plan flow by replacing a broken SSE-as-JSON fetch with a `triggerImprovePending` store signal so the layout's working `handleImprove()` fires instead. Completed research steps 1 and 2 for plan-execution-mode naming, producing two research documents and recommending `mode: mentor | guided | autonomous`. Tagged v0.2.4 as a Phase 2 milestone.

## Decisions Made

- **Remove custom Ollama picker**: OpenCode already has a full model selector with all cluster/local models configured. DocWright routes AI calls through OpenCode — switching models there applies automatically. No separate backend picker needed.
- **AI model picker via toolbar**: `/api/opencode-model` GET reads live model list from OpenCode, PUT writes to `opencode.json`. Picker loads lazily on first click, shows full 112-model dropdown.
- **`triggerImprovePending` store pattern**: `triggerImproveOnSave` was calling `.json()` on an SSE stream (silent fail). New pattern: page sets the store, layout reacts and calls `handleImprove()` (the working SSE reader).
- **`handleCreatePlan` → `?from=proposal`**: Changed from `?new=1` so new plans created from proposals auto-improve on load.
- **Phase assignment naming**: recommended `mode: mentor | guided | autonomous` (replaces `automated: off | guided | full`). Research docs in `research/`.
- **v0.2.4 tag**: Phase 2 milestone cut with all session work included.
