## Summary

Implemented live file change detection via Server-Sent Events (SSE) for the docwright
SvelteKit Web UI. The `/api/watch` endpoint uses `fs.watch` recursive to emit `filechange`
events on any `.md` file create/update/delete. The client auto-reloads content and
refreshes the file tree without manual refresh.

## Completed

- Created `/api/watch` SSE endpoint (`src/webui/src/routes/api/watch/+server.ts`)
- Created shared `fileChanged` Svelte writable store (`src/webui/src/lib/fileChanges.ts`)
- Connected layout `+layout.svelte` to SSE on mount, pushes events into store
- Updated `FileTree.svelte` to subscribe to store, debounce 200ms, re-fetch tree
- Updated `[...path]/+page.svelte` to reload content on matching file changes (skips when editing)
