Fixed the entire theming pipeline by moving CSS variables to `app.html` and using `:global()` inside component styles. Fixed multiple UX lifecycle gaps: Web UI commits now carry `HUMAN_APPROVED=1`, Certify Tests button saves immediately, Complete button now fully archives the plan, and a Run Tests gate must pass before Complete appears. Tagged v0.2.1.

## Decisions Made

- CSS variables must live in `app.html` (static, before any Svelte JS) — not in `<svelte:head>`
- `:global(html[data-theme="light"]) .scoped-class` is the correct pattern for component light-mode overrides (Svelte appends scope hash)
- `brand/theme.css` must use `var()` — any hardcoded hex there overrides everything with `!important`
- Web UI commits implicitly carry `HUMAN_APPROVED=1` (browser tool = human by definition); Phase 2 replaces with Forgejo OAuth
- Complete button: calls `/api/lifecycle/transition-completed` → archives plan → redirects to `/status`
- Run Tests must pass before Complete appears; step changes reset `tests_defined: false` automatically
- Automated test lifecycle captured as proposal `automated-test-lifecycle.md`
