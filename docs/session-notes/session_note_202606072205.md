## Summary

Built and refined the phase gate review system on the status page. The system detects when a phase has been gated/completed and prompts the human to review remaining phase plans before activating the next one. Fixed two display bugs: the active phase card was showing Phase 4 (max across all plans) instead of the actual in-progress phase, and `phasePlans` was matching any plan with "phase-N" in its path rather than only top-level phase overview plans. Implemented the "Done reviewing" button with a gate: it only becomes available when all non-completed plans tagged with that `phase:` number are finished — showing "⏳ N open plans" otherwise.

## Decisions Made

- `activeWorkCount` counts ALL non-completed, non-canceled plans with a matching `phase:` field — including the overview plan itself. This is intentional: Phase 2's review button should remain blocked while `phase-2-foundation.md` is still in-progress.
- Phase card `currentPhase` uses the in-progress/approved phase overview plan's phase number, not `max(phase:)` across all plans.
- `phasePlans` filter uses `path.basename(p)` matching `/^phase-\d/` to exclude non-overview plans like `bundle-lifecycle-gates-phase-2.md`.
- Phase review banner CTA is conditional: shows "Phase N is already active ✓" when status is in-progress; shows "click Start in Properties panel" when still draft.
