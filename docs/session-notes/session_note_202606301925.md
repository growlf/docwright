# Session Note: 2026-06-30 — devdw SSH connectivity fix

**Date:** 2026-06-30
**Author:** NetYeti @ phoenix
**Focus:** devdw SSH connectivity fix

## Summary

Diagnosed why the devdw remote-session script (from 2026-06-29) could no longer connect to gemini (10.10.0.201): ssh port 22 was reachable but the handshake hung at SSH2_MSG_KEX_ECDH_REPLY. Root cause was OpenSSH's default post-quantum hybrid KEX (sntrup761x25519-sha512) sending oversized init packets that were silently dropped somewhere on the path, a PMTU black hole. Forcing curve25519-sha256 connected instantly, confirming the diagnosis. Fixed by adding a KexAlgorithms exclusion to the matching Host block in ~/.ssh/config (outside this repo), and verified both plain ssh and devdw --list now connect cleanly. No repo changes were needed.

## Decisions Made

- Disable sntrup761x25519-sha512@openssh.com specifically for the 10.10.0.201/cluster-llm/gemini host block rather than globally, to keep PQ KEX available for hosts that don't hit this path issue
