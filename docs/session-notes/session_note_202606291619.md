## Summary

Created `devdw`, a shell script at `~/.local/bin/devdw` that provides a resilient, persistent connection to the gemini development server. The script prefers mosh over SSH for connection resilience (survives roaming, sleep, packet loss) and uses a named tmux session so Claude Code sessions survive disconnects. Resolved several setup issues on gemini: the `en_US.UTF-8` locale was commented out in `/etc/locale.gen` (mosh requires UTF-8), Claude Code was installed via npm but not on the system PATH, and the work directory was set to the wrong name (DocWrite vs DocWright).

## Decisions Made

- Use `mosh + tmux` as the persistence stack (not screen); mosh handles connection resilience, tmux handles session persistence
- Script auto-detects mosh availability and falls back to SSH — supports restricted networks (hotels, corporate UDP blocks) via `--ssh` flag
- Claude Code symlinked to `/usr/local/bin/claude` so it is available regardless of shell interactivity level
- Script named `devdw` (not `gemini`, which conflicts with an existing command)
