# Session Note: 2026-06-07 — CI/CD build monitoring, tag policy, Web UI watch

**Date:** 2026-06-07
**Author:** NetYeti @ phoenix
**Focus:** CI build failures, tag enforcement, automated CI watch

## Summary

Session opened with three consecutive CI build failures on the main branch. The root cause was `__dirname` not available in ES module scope in the new `test/compat/cross-tool.test.ts` — fixed with `fileURLToPath(import.meta.url)`. CI was then hardened: the workflow trigger was narrowed from every branch push to `push: tags: v0.*.*` and `workflow_dispatch` only, making tagged releases the sole CI/CD gate. A three-layer automated CI watch system was built: a Claude Code `PostToolUse` shell hook, an OpenCode behavioral rule, and an `npm run release:tag` script. Finally, the Web UI gained a native SSE-based CI watch panel in `GitPanel.svelte` that streams live run status after every tag push. The `v0.x.x` policy is now enforced at the API layer, in the CI trigger, in the watch scripts, and documented in `policies/core/versioning.md` with an explicit update checklist for future major milestone releases.

## Decisions Made

- CI only fires on `v0.*.*` tags and `workflow_dispatch` — no more per-push/PR runs
- "release" bump type removed from GitPanel (would generate policy-violating `v1.0.0`)
- CI watch is pure shell + `gh` CLI — no AI tokens consumed by the monitoring itself
- OpenCode gets a behavioral rule (not a hook) until OpenCode's hook API stabilises
- `npm run release:tag` is the canonical cross-tool path for pushing a release tag
- `v0.*.*` restriction is temporary and policy-gated; updating it is a checklist item in any future major milestone release plan

## Architecture State

CI is now gated on version tags only. Three enforcement layers exist for CI watch:
1. Claude Code: `PostToolUse` hook in `.claude/settings.json` → `scripts/claude-tag-push-watch.sh`
2. OpenCode: `.opencode/rules/ci-watch-on-tag-push.md` (behavioral, until hooks ship)
3. Any tool: `npm run release:tag` → `scripts/release-tag.sh`
4. Web UI: `/api/git/ci-watch` SSE endpoint → `GitPanel.svelte` live status panel

Tag creation is now validated to `v0.x.x` at the API layer (`/api/git/tag`) and the `release` bump type (which would produce `v1.0.0`) is removed from the UI. Policy documentation in `versioning.md` includes the CI trigger as a required update item in any future major release plan.

Pre-existing TS2339 bug in `watch/+server.ts` fixed (filename: never → String(filename)).
