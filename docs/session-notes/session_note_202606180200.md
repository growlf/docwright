## Decisions Made

- `$effect` is wrong for D3 graph rendering — reactive tracking fired multiple times per data load, restarting the simulation before nodes could spread. All graph rebuilds now use explicit `buildGraph()` calls triggered by explicit filter/load events.
- Knowledge graph uses keyword-coverage engine (not Jaccard) for pre-creation duplicate checks — Jaccard needs full document length to score meaningfully; 1–3 sentence descriptions score near zero.
- Consumed proposals (converted to plans) hidden by default in graph — they clutter the graph since the plan node already represents that work. "Show consumed proposals" checkbox makes them opt-in.
- Phase 3 vault write API completely replaces direct `fs.renameSync` in all three endpoints that moved files — no new file moves should bypass it.
