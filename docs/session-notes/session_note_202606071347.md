## Summary

A long session covering a wide range of housekeeping, bug fixes, new features, and
strategic recalibration. The relationship engine bug (block-style YAML tags silently
dropped) was fixed — collation now actually works. A full suite of plan/proposal
lifecycle flow gaps were found and repaired (draft→approved with no button, Plan →
navigating to existing plan instead of creating orphans, consumed_by not set on
approve, etc). The AI Review (⚡) and AI Improve (✨) features were fully implemented
and live-tested against OpenCode. The status page was upgraded with priority sorting
and dependency display. PROJECT.md was recalibrated from v0.8 to v0.9 — phases now
reflect the actual build order (Web UI-first) rather than the original extension-first
plan. A major cleanup of the proposals and plans backlog was completed (36 orphaned
proposals linked to completed plans, deferred proposals audited). Three new research
proposals were captured.

## Decisions Made

- `research/` is now a formal lifecycle stage in the methodology (critical priority,
  Phase 2 target) — captured in `proposals/research-stage-methodology.md`
- `automated: off` naming is wrong — should be `mentor` or similar; research proposal
  created for plan execution modes
- Approve on proposal auto-creates a plan going forward; "Approved — awaiting plan"
  section is now a deliberate backlog, not a bug tracker
- The status page priority sort order: critical → high → medium → low → unset
- `untrack()` is the correct fix for Svelte 5 `$effect` dependencies that must not
  re-trigger on state they modify
