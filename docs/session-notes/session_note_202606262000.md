## Summary

Overhauled the UI layout view-container refactor plan from 14 to 18 steps with a
new Phase 0 (API Contract). The plan now has a unified bridge API, full lifecycle
callbacks, a priority-based right panel ownership model, error boundary requirement
before any view extraction, TypeScript type declarations, lazy bundle loading, and
a mobile VC strip. Step 1 is fully complete (unified `window.__docwright` bridge +
`plugin-api.d.ts`). Step 2 is partially complete — `ViewContainerMount.svelte` is
written and pushed, but the `+layout.svelte` wiring remains for next session.

## Decisions Made

- Dual globals (`window.__dw_plugins` Map + `window.__docwright_host`) unified into
  `window.__docwright` — single entry point for all plugin/shell interaction.
- `mount(el)` replaces `mountSidebar()` timing hack — shell passes the element directly.
- Right panel: priority model (VC claim > active doc Properties) replaces binary ownership.
- Error boundary (`ViewContainerMount`) required BEFORE any view extraction starts.
- `bridge.emit()` for shell signalling reserved for core plugins only (not external).
- Mobile VC strip lives inside the left panel Panel slot (not in Panel.svelte itself).
- `window.__docwright_host` kept as backward-compat shim (with `setRightPanel`/`clearRightPanel` aliases); removed in Step 11.
