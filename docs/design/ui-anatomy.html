Component Reference
1
Toolbar
.app-toolbar
Always-visible top bar. Contains hamburger toggle, brand/home link, + New menu, AI model picker, and right-panel toggle. DocWright chrome — plugins do not control it.
2
Activity Bar
.activity-bar
The leftmost vertical strip. The single source of truth for which view is active. Always visible; always shows which icon is selected. DocWright-controlled chrome — persistent across all views.
3
View Container Icon
.act-btn
A single icon slot in the Activity Bar. Each registered view (Files, Git, a plugin) gets one. Active state shown with left accent border. Plugin icons appended dynamically from /api/plugins.
4
Left Panel
.panel-left / #erp-images-sidebar-root
Content area beside the Activity Bar. When a DocWright view is active: DocWright owns it (file tree, git panel, etc.). When a plugin view is active: plugin owns the entire panel — no header, no chrome, full height.
5
Sidebar Header
.sidebar-header
Label bar at top of Left Panel. Shows the view name and optional actions (e.g. + for Files view). Present in DocWright native views; absent when plugin owns the panel — the plugin provides its own top-of-panel chrome.
6
Left Panel Content
mounted by plugin or DocWright view
The scrollable content below the Sidebar Header. DocWright renders file trees, panels, tag lists. Plugins register a mountSidebar() function in window.__dw_plugins and DocWright calls it when the view switches.
7
Main Content Area
#content / SvelteKit routes
The center pane. Driven by URL routing. DocWright renders markdown documents, status pages, graph views. Plugins contribute routes under /plugin/[name] and encode sub-state as query params (e.g. ?uc=helpdesk&tag=v16.1.0).
8
Right Panel
.panel-right / #erp-images-right-panel-root
Context-sensitive panel. When a DocWright view is active: shows Properties / Related / Review tabs. When a plugin view is active: plugin owns the entire panel — tab bar replaced by a plugin header, content populated via bridge.setRightPanel(html).
9
Right Panel Header / Tab Bar
.right-tab-bar / .plugin-right-header
DocWright native views: tab bar with Properties, Related, Review, Improve, Execute. Plugin views: replaced by a plain header showing the plugin's display name. Tabs only appear when DocWright owns the panel.
10
Right Panel Content
PropertiesPane / plugin HTML
DocWright: document frontmatter fields, related documents, AI review output. Plugin: any HTML provided to bridge.setRightPanel(). Should match DocWright's visual language (use var(--fg), var(--muted) etc.).
11
Footer
.app-footer
Bottom strip. Contains attribution, theme picker. Future: Settings link. DocWright chrome — plugins do not control it. Persisted across all views.
⚡
Chat Toggle
.chat-toggle
Fixed floating button above the footer. Opens/closes the AI chat panel which springs up from the bottom of the Main Content Area. DocWright chrome — always present regardless of active view.
Panel Ownership Rules
🟢 Plugin-owned when active
  • Left Panel — full height, no DocWright header
  • Left Panel Content — via mountSidebar()
  • Right Panel — full height, plugin header replaces tab bar
  • Right Panel Content — via bridge.setRightPanel(html)
  • Main Content routes under /plugin/[name]
🔵 Always DocWright-controlled
  • Toolbar — hamburger, brand, + New, model picker
  • Activity Bar — icon registration, active state
  • Footer — attribution, theme picker
  • Chat toggle and chat panel
  • Toast and notification system
Plugin Contract Quick Reference

Registration (bundle.js)

KeyDescription
window.__dw_pluginsMap of registered plugins. Layout reads from this.
.set('name', obj)Register with { mountSidebar }
mountSidebar()Called by layout via requestAnimationFrame when leftView switches to this plugin

Bridge (window.__docwright_host)

MethodDescription
.toast(msg, ms)Show a toast notification
.notify({type, title, message})Persistent notification panel
.setRightPanel(html, label)Populate right panel, opens it
.clearRightPanel()Empty right panel on teardown

URL State Convention

URL patternMeaning
/plugin/[name]Plugin main content — help/welcome page
/plugin/[name]?uc=helpdeskUse case selected, no specific tag
/plugin/[name]?uc=helpdesk&tag=v16.1.0Specific tag selected — shareable, survives refresh
(future) /plugin/[name]/path/to/thingPath-based routing for cleaner URLs — opt-in per plugin via plugin.json

leftView persistence: stored in localStorage('dw-left-view') — survives page refresh. When navigating to a non-plugin URL while a plugin panel is active, the active view resets to 'files'.
