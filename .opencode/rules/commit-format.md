# Rule: Commit Message Format

All commit messages MUST follow this format:
```
<type>: <description>
```

Valid types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`

The description is a brief summary in lowercase (no trailing period).

Examples:
- `feat: add dns record management skill`
- `fix: correct template variable resolution`
- `docs: update agent lifecycle documentation`
