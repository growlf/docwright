# Carryover: still-valid slices from the canceled milestone-discipline plan

## Summary

`plans/adopt-milestone-driven-roadmap-discipline.md` (created 2026-07-10) was **canceled**
2026-07-13 as overtaken-by-events — most of it shipped this session, its versioning wave was
*reversed* by the shipped policy, and two steps collide with the in-flight authz redesign
(full currency review: [[docs/authz-model-hardening-review]]'s sibling analysis + that plan's
cancellation reason). Per capture-deferred-ideas, this proposal preserves the slices that are
**still valuable and un-done**, re-scoped to current reality (v0.5.3; issues are
GitHub-canonical; patch is per-release; `HUMAN_APPROVED` is being retired). Split each into its
own plan when prioritized.

## Already covered — no re-capture needed (just un-consumed by the cancellation)

These retain **standalone approved proposals** in `proposals/approved/`; the cancellation
orphaned their `consumed_by` pointer but the ideas are intact. Re-plan fresh when prioritized:
- **Structured plan steps** (`steps:` YAML schema, dual-write, validation) — but **rebase on
  the GH-pivot**: step status derived from issues now reads **GitHub Issues**, not local
  `tracked_by:`/`step_issues.ts`.
- **`automated:` → `mode:` field rename.**
- **Roadmap sequencing enforcement** (partial — see the living-roadmap slice below).

## Still-valid slices to re-capture here

### 1. Testing tiers (was W4)
`docs/testing-tiers.md` — tier 1 automated (unit + CI), tier 2 dogfood-smoke, tier 3 manual UI
checklists — plus a `testing_tier` (list) plan-frontmatter field enforced by
`policies/frontmatter-validate/check.ts` **and** its `scripts/pre-commit.sh` mirror, with a
one-time backfill of active plans. The plan template's Testing Plan section asks which tiers apply.

### 2. AGENTS.md onboarding surface (was W6)
A bounded generated zone (`<!-- START/END_STATUS -->`) at the top of AGENTS.md: current version,
done / in-progress / next, reading order — regenerated by `session:end` + pre-commit, with a CI
**consistency** check (header version == VERSION, active plan == get_status). Plus a cold-start
fresh-agent verification run against a pre-recorded answer snapshot. (The two end-session
protected-main bugs that gated this were addressed this session — verify before relying on them.)

### 3. Living roadmap renderer (was W5)
Generate `docs/roadmap.md` from plan state (extend `src/dispatch/roadplan.ts` /
`generate-roadplan.ts`), with a required CI drift-guard (`roadplan --check`). **Rebase on the
GH-pivot:** the milestone↔GitHub-milestone mirror and the two-axis model (milestone = release
bucket, phase = versioning backbone) must reflect that dev issues are now GitHub-canonical.
Complements the approved roadmap-sequencing proposal.
