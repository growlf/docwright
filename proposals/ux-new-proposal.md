## Problem

This is another UX/UI proposal.

I want to be able to simply click a button to open a new proposal form that pre-populates defaults, supplies pick-lists for things like known users to assign it to, etc. 

## Proposed Solution

When editing proposals, it should also use the same form as much as is reasonable.  When generating the proposal, the "first-time" uses the main pane with the form at the top so that the user sees it and understands that it MUST be filled out. Then, when editing the proposal later, the main pane is the generated doc, and the form is in the right-side pane.
